Performance Measurement Tool

The performance measurement tool enables testing of the performance of Intel® Image Processing Library functions on your own platform. The output is .csv, (Comma-Separated Value), a text-based format readable by Microsoft* Excel*.

The following are the command-line parameters of this tool. Alternatively, a batch file is provided that will use the most common flags to produce a performance measurement.

-A

only generates a file summary.csv, containing the information on presence of the tests

 
-B string

batch mode, substitute all reads from CON by 'string'

 
-B

accept default (simulate <CR>)

 
-B";>"

accept default and write to default sections

 
-B";>[.i486];>[.p5]"-

accept default, write to extended sections 

-B";>[nsp]"

accept default, write to absolute section 

-B -D"*<"

data from file, if no data, simulate <CR>

 
-B"!e" -D"*<"

data from file, if no data, then exit

 
-c name

class, apply command to tests of class 'name'

 
-c ?

list all classes of tests

 
-c Algorithm

run all tests with class 'Algorithm'

 
-C name

name of .csv file for performance numbers

 
-d

DEBUG mode, activate tstWrite(TW_DEBUG,...)

 
-D string

define symbols through 'string' 

-D "a=1;b=4.3;c=abcd.ext"
   
-D "?h;?l;!e"

print test help, listing and exit

 
-D "*<"

?Read from section [funcname.testclass.testbody]

 
-D "*<[]"

?Read from section [funcname.testclass.testbody]

 
-D "*<[.p5]"

?Read from ext section [funcname.testclass.testbody.p5]

 
-D "*<a.ini[nsp]"

?Read from a.ini, absolute section [nsp]

 
-f name

apply command to function with name 'name'

 
-f ?

show functions list

 
-f nspsAdd

run all tests for function 'nspsAdd'

 
-h

short help

 
-H

full help (this message)

 
-i name

set current ini file name

 
-l name

listing(TW_LST), output to file 'name', default *.lst

 
-L n

loop, repeat each test 'n' times

 
-m n

more mode, n lines per page, if no 'n', more off,

 
-R result

print to .sum (-s) only tests with result=OK/FAIL

 
-r n

RERUN Mode

 
-r

force RERUN mode, activate tstWrite(TW_RERUN,...)

 
-r 2

if test FAIL repeat test 2 times in RERUN Mode

 
-s name

summary (TW_SUM), only test results, default *.sum

 
-S name

source, apply command to tests in file 'name'

 
-S ?

show all source files in test system

 
-S abcd.c

run all tests in source file 'abcd.c'

 
-t name

testname, apply command to functions with testname 'name'

 
-t ?

show testnames list

 
-t Test1

run all tests with name 'Test1'

 
-T name

testbody, apply command to testbody 'name'

 
-T ?

show testbodies list

 
-T tstbody

run all tests with bodyname 'tstbody'

 
-- string

any user defined string, key -- is equiv to -Z
